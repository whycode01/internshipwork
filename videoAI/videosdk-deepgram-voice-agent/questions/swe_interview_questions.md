# Software Engineering Interview Questions

This file contains comprehensive software engineering interview questions covering various topics.

## Question 1
**Topic:** Data Structures
**Question:** Explain the difference between arrays and linked lists. When would you choose one over the other?
**Type:** Concept
**Difficulty:** Easy

## Question 2
**Topic:** Algorithms
**Question:** What is the time complexity of binary search? Can you implement it recursively and iteratively?
**Type:** Implementation
**Difficulty:** Medium

## Question 3
**Topic:** System Design
**Question:** How would you design a URL shortener service like bit.ly? Consider scalability and performance.
**Type:** Design
**Difficulty:** Hard

## Question 4
**Topic:** Object-Oriented Programming
**Question:** Explain the four pillars of OOP: Encapsulation, Inheritance, Polymorphism, and Abstraction. Provide examples.
**Type:** Concept
**Difficulty:** Medium

## Question 5
**Topic:** Database Design
**Question:** What's the difference between SQL and NoSQL databases? When would you use each?
**Type:** Knowledge
**Difficulty:** Medium

## Question 6
**Topic:** Coding Challenge
**Question:** Write a function to reverse a string without using built-in reverse methods. What's the time and space complexity?
**Type:** Coding
**Difficulty:** Easy

## Question 7
**Topic:** Web Development
**Question:** Explain the difference between REST and GraphQL APIs. What are the pros and cons of each?
**Type:** Technical
**Difficulty:** Medium

## Question 8
**Topic:** Testing
**Question:** What's the difference between unit testing, integration testing, and end-to-end testing? Give examples.
**Type:** Knowledge
**Difficulty:** Easy

## Question 9
**Topic:** Concurrency
**Question:** Explain the difference between multithreading and multiprocessing. When would you use each approach?
**Type:** Concept
**Difficulty:** Hard

## Question 10
**Topic:** Problem Solving
**Question:** Given an array of integers, find two numbers that add up to a target sum. Optimize for time complexity.
**Type:** Coding
**Difficulty:** Medium
